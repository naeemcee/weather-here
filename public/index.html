How it works
  • Developed as a full-stack solution; a client side and a node server back-end.
  • Client gets geo coordinates via browser navigator call and sends the location data to backend to obtain weather and air-quality data.
  • Server makes two API calls; for weather and for Air Quality data respectively, and sends it back to client.
  • Client plots the location on a Leaflet.js map and displays the weather and air-quality on the map.
  • If the client clicks the 'add to log' button, server adds the weather and air-quality data to the logbook database, using Node-Nedb library.
  • Client can ask to see the logbook entries by clicking on the 'View log' button.
  • Database entries are sent to client where it is plotted on a Leaflet map with tooltip loaded with the details.
